# PRD：AI 媒体创作 App

## 项目信息

- **Language**: 中文
- **Programming Language**: Vite + React + MUI + Tailwind CSS
- **Project Name**: `aigen_media`
- **原始需求**: 开发一个 AI 生成图像和视频的 App，支持文生图（Text-to-Image）和文生视频（Text-to-Video）核心能力

---

## 产品定义

### Product Goals

1. **降低创作门槛**：让零设计/视频制作经验的用户，通过自然语言描述即可生成高质量视觉内容
2. **提升创作效率**：从想法到成品，将传统数小时的设计/剪辑流程压缩至分钟级
3. **构建创作闭环**：从灵感输入 → 内容生成 → 预览调整 → 导出分享，提供一站式体验

### User Stories

1. **As a** 社交媒体运营，**I want** 输入一句话描述即可生成配图，**so that** 我能快速为每条推文配上原创图片，不再依赖素材库
2. **As a** 短视频创作者，**I want** 输入场景描述生成3-5秒视频片段，**so that** 我能快速制作创意短片或视频素材
3. **As a** 产品设计师，**I want** 通过文字描述快速生成概念图，**so that** 我能在早期探索阶段高效迭代视觉方案
4. **As a** 个人用户，**I want** 在手机上随时生成有趣图片/视频并分享，**so that** 我能用 AI 创作表达创意
5. **As a** 内容团队负责人，**I want** 统一管理团队生成历史，**so that** 团队成员能复用和追溯创作成果

## 技术规范

### Requirements Pool

#### P0 — Must Have（MVP 核心）

| ID | 需求 | 说明 |
|----|------|------|
| P0-01 | 文生图 | 用户输入文字 Prompt，调用 AI 模型生成图片，支持选择尺寸（1:1 / 16:9 / 9:16） |
| P0-02 | 文生视频 | 用户输入文字 Prompt，调用 AI 模型生成短视频（3-5秒），支持选择分辨率 |
| P0-03 | Prompt 输入界面 | 提供文本输入框、风格/尺寸选择器、生成按钮，支持中英文输入 |
| P0-04 | 生成结果展示 | 展示生成的图片/视频，支持全屏预览、下载保存到本地 |
| P0-05 | 生成历史 | 记录用户所有生成任务及结果，支持按时间倒序浏览 |
| P0-06 | 用户认证 | 支持邮箱/手机号注册登录，保障数据隔离与安全 |

#### P1 — Should Have（增强体验）

| ID | 需求 | 说明 |
|----|------|------|
| P1-01 | 图生图 | 上传参考图片 + 文字描述，生成风格/内容相似的新图片 |
| P1-02 | 图生视频 | 上传静态图片，AI 将其转化为动态视频 |
| P1-03 | Prompt 模板 | 提供预设 Prompt 模板（如"赛博朋克城市"、"水彩花卉"等），降低创作门槛 |
| P1-04 | 生成参数调节 | 支持调节关键参数（如创意度/引导系数、生成步数等），满足进阶用户需求 |
| P1-05 | 结果收藏与标签 | 用户可收藏生成结果，添加自定义标签，支持按标签筛选 |
| P1-06 | 批量生成 | 单次任务生成多张图片（2/4张），提升选择空间 |

#### P2 — Nice to Have（锦上添花）

| ID | 需求 | 说明 |
|----|------|------|
| P2-01 | 视频编辑 | 对生成的视频进行简单剪辑（裁剪时长、添加文字/滤镜） |
| P2-02 | 社区分享 | 用户可公开分享作品，浏览和点赞他人作品 |
| P2-03 | 团队协作 | 多人共享工作空间，统一管理生成历史与额度 |
| P2-04 | API 开放 | 提供 REST API，允许第三方接入生成能力 |
| P2-05 | 本地模型支持 | 支持用户本地部署模型，离线生成内容 |

## UI Design Draft

### 整体布局

采用侧边导航 + 主内容区经典布局，移动端切换为底部 Tab 导航。

### 页面清单

#### 1. 首页 / 创作台（核心页面）

```
┌─────────────────────────────────────────────┐
│  Logo    [创作台]  [历史]  [设置]    Avatar  │  ← 顶栏
├─────────────────────────────────────────────┤
│                                             │
│  ┌─────────────────────────────────────┐    │
│  │  🎨 文生图  |  🎬 文生视频           │    │  ← 模式切换 Tab
│  └─────────────────────────────────────┘    │
│                                             │
│  ┌─────────────────────────────────────┐    │
│  │  描述你想要的画面...                 │    │  ← Prompt 输入框（多行）
│  │                                     │    │
│  └─────────────────────────────────────┘    │
│                                             │
│  风格: [写实] [动漫] [油画] [赛博朋克] ...  │  ← 风格快捷选择
│                                             │
│  尺寸: [1:1] [16:9] [9:16]                 │  ← 尺寸选择器
│                                             │
│  ┌─────────────────────────────────────┐    │
│  │          ✨ 生成                      │    │  ← 主操作按钮
│  └─────────────────────────────────────┘    │
│                                             │
│  💡 试试: "夕阳下的海边小屋，水彩画风格"    │  ← 灵感提示
│                                             │
└─────────────────────────────────────────────┘
```

#### 2. 生成结果页

- 生成中：展示进度条 + 预计等待时间
- 生成完成：展示结果图片/视频播放器
- 操作区：下载、收藏、重新生成、分享
- 底部：展示该次生成的完整 Prompt 和参数

#### 3. 历史记录页

- 左侧/顶部：筛选器（全部/图片/视频/收藏）
- 主区域：瀑布流/网格展示历史生成结果
- 每项：缩略图 + Prompt 摘要 + 时间 + 状态标签
- 点击进入详情，可查看大图/播放视频、查看原始参数、重新生成

#### 4. 设置页

- 账户信息与认证
- 生成默认参数（默认风格、默认尺寸）
- 存储管理（本地缓存清理）
- 关于与帮助

### 交互要点

- 生成任务提交后自动跳转结果页，支持后台生成（用户可切换到其他页面）
- 历史记录实时更新生成状态（排队中 → 生成中 → 完成/失败）
- 移动端适配：输入区占满宽度，操作按钮固定底部

## Open Questions

1. **AI 模型接入方式**：是调用第三方 API（如 Stability AI / Runway / Pika 等），还是自建模型服务？不同方式影响成本、延迟和合规策略
2. **计费模式**：是否需要额度/积分系统？免费用户与付费用户的生成能力如何区分？
3. **目标平台**：当前定位是 Web App、桌面 App（Electron）、还是移动端优先？技术栈为 Vite+React，优先 Web 版是否合理？
4. **内容审核**：生成的图片/视频是否需要内容安全审核（如 NSFW 过滤）？审核策略是前置（拦截 Prompt）还是后置（过滤结果）？
5. **视频生成时长上限**：MVP 阶段文生视频的时长上限是多少？3秒、5秒还是更长？这直接影响成本和等待时间
6. **多语言支持**：是否需要支持非中文/英文的 Prompt 输入？AI 模型对中文 Prompt 的理解能力如何？
7. **数据存储**：生成结果存储在云端还是本地？是否需要跨设备同步？
